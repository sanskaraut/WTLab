1. Problem Title: Product Inventory Management using Spring Boot and MongoDB

2. System Overview:
This project demonstrates the integration of a Java-based backend (Spring Boot) with a NoSQL database (MongoDB). It provides a RESTful interface to manage a catalog of products, allowing for flexible data storage without rigid schemas.

3. Technologies Used:
- Java / Spring Boot (Framework)
- Spring Data MongoDB (ORM/ODM)
- Maven (Build Tool)

4. Working Flow:
Client sends JSON via POST → Spring Controller receives object → MongoRepository saves it to MongoDB → ID is auto-generated by the database → Client can later retrieve all items via a GET request.

5. Code Explanation:
- @Document: Marks the Java class as a MongoDB collection.
- MongoRepository<Product, String>: A Spring Data interface that provides ready-to-use methods like `save()`, `findAll()`, and `deleteById()`.
- @RestController: Combines `@Controller` and `@ResponseBody`, simplifying the creation of RESTful web services.
- @RequestBody: Tells Spring to map the incoming JSON directly into a Java object.

6. Key Concept:
NoSQL Integration: Using an abstraction layer (Spring Data) to communicate with a document-oriented database, simplifying complex data operations into simple method calls.
